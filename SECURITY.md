# Kebijakan Keamanan

## Melaporkan Kerentanan

Untuk melaporkan kerentanan, silakan [laporkan secara pribadi melalui tab Keamanan](https://github.com/nlfts/lions/security/advisories/new) pada repositori GitHub yang benar (lihat [dokumentasi](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability#privately-reporting-a-security-vulnerability)). Jika hal itu tidak memungkinkan, silakan kirim email ke **team@nlfts.dev** sebagai gantinya.

Semua kerentanan keamanan akan segera diverifikasi dan ditangani.

Meskipun penemuan kerentanan baru jarang terjadi, kami juga merekomendasikan untuk selalu menggunakan versi Vuejs, Java dan dependensi lainnya yang terbaru dengan memelihara file lock (`yarn.lock`, `package-lock.json` dan `pnpm-lock.yaml`) untuk memastikan aplikasi Anda tetap seaman mungkin.
